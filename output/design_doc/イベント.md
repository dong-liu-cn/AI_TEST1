## 3. 各イベント詳細

### EVT-001: 初期表示

| 項目 | 内容 |
|---|---|
| トリガー | 画面ロード |
| 処理概要 | 画面の初期値を設定する |

**処理フロー:**

1. 年月に「運用日の年月（YYYY/MM）」を設定する
2. 支払確認書処理ステータスのチェックボックス（未完了・完了）をすべてチェック状態にする
3. 発行状況のチェックボックス（未発行・発行済・仮公開・公開済）をすべてチェック状態にする
4. 明細部は空状態で表示する
5. 明細操作部の「実行」ボタンは非活性とする（TBD）

---

### EVT-002: 検索

| 項目 | 内容 |
|---|---|
| トリガー | 「検索」ボタン押下 |
| 利用タイミング | ① 検索時 |
| 利用者 | 調達管理部門管理者 / 調達管理部門作業者 |
| 処理概要 | 検索条件に該当する仕入先の情報を取得して一覧を表示する |

**処理フロー:**

1. **バリデーション**
   - 年月が入力されていること（必須チェック）
2. **DB検索**
   - 指定条件で支払確認書発行状況テーブルを検索
   - レコードが存在しない仕入先は「未発行」として扱う
   - 何らかの支払いが存在する仕入先のみ表示（0円除く）
3. **一覧表示**
   - 取得結果を明細部に表示する
   - 出力順：組織コード、仕入先コード
4. **後処理**
   - 仕入先に対する支払確認書の発行状況を取得し、明細一覧に反映する

**補足:**
- ページング制御が有効になる

### EVT-007: 支払確認書仮公開

| 項目 | 内容 |
|---|---|
| トリガー | 「実行」ボタン押下（処理区分：仮公開を選択した状態） |
| 利用タイミング | ④ 支払確認書仮公開時 |
| 利用者 | 調達管理部門管理者 / 調達管理部門作業者 |
| 処理概要 | 明細部の選択欄にチェックがある仕入先の支払確認書を仮公開する |

**処理前提:**

| No | 前提条件 | エラー時のメッセージ |
|---|---|---|
| 1 | 明細部の選択欄にチェックがあること | — |
| 2 | デポに対して「発行済」の支払確認書であること | 「支払確認書が未発行の仕入先が選択されています。」 |
| 3 | 処理区分で「仮公開」が選択されていること | — |
| 4 | WEBデポを使用していない仕入先が選択されていないこと | （エラーメッセージ TBD） |

**処理フロー:**

1. **バリデーション**
   - 選択仕入先が1件以上あること
   - 選択仕入先がすべて「発行済」であること
   - 選択仕入先がすべてWEBデポを使用していること
   - バリデーションNGの場合はエラーメッセージを表示し、処理を中断
2. **DB更新**
   - 支払確認書発行状況テーブルの公開フラグ = 1（仮公開）に更新
3. **メール送信**
   - WEBデポ担当者に「仮公開」された旨のメールを送信
4. **完了処理**
   - 明細の再検索を行う（EVT-002 と同条件で再実行）

### EVT-009: Excel出力

|---|---|

**処理前提:**
- 検索条件において必須項目（年月）が入力されていること

**処理フロー:**

   - 出力順：検索結果一覧の表示に準ずる（組織コード、仕入先コード順）
   - 詳細は別紙「共通設計書」を参照
4. **完了処理**
   - 出力処理完了の確認メッセージを表示する

|---|---|
| トリガー | カレンダーアイコン押下 |
| 処理概要 | 年月入力欄に対応するカレンダーを表示する |

**処理フロー:**

## 4. イベント処理フロー全体図

```
[画面ロード]
    └─ EVT-001: 初期値設定（年月=運用日年月、チェックボックス全ON）

[検索条件入力]
    ├─ EVT-003: 組織検索サブ画面 → 組織コード/名 設定
    ├─ EVT-004: 仕入先検索サブ画面 → 仕入先コード/名 設定
    ├─ EVT-005: 在庫グループ検索サブ画面 → 在庫グループ/名 設定
    ├─ EVT-006: 法人検索サブ画面 → 法人コード/名 設定
    └─ EVT-011: カレンダーアイコン → 年月 設定

[検索実行]
    └─ EVT-002: 検索 → 明細部に一覧表示

[明細操作]
    ├─ EVT-007: 実行（仮公開）→ DB更新 → メール送信 → 再検索
    └─ EVT-008: 実行（公開）→ DB更新 → メール送信 → 再検索

[出力]
    └─ EVT-009: Excel出力 → ファイルダウンロード → 完了メッセージ

[ページング]
    └─ EVT-010: 先頭頁/前頁/後頁/最終頁 → ページ切替
```
