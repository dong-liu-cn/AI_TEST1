### 3-2. 支払確認書仮公開

| 項目 | 内容 |
|---|---|
| API論理名 | 支払確認書仮公開 |
| 物理エンドポイント | TBD |
| HTTPメソッド | TBD（推定：POST または PUT） |
| トリガー | ④実行ボタン押下（処理区分：仮公開） |

**リクエストパラメータ（想定）**

| パラメータ名 | 必須 | 説明 |
|---|---|---|
| 仕入先コードリスト | ○ | チェックした仕入先の一覧 |
| 年月 | ○ | 処理対象の年月 |

**処理前提（バリデーション）:**
- 明細部の選択欄にチェックがあること
- デポに対して「発行済」の支払確認書であること
- 処理区分で「仮公開」が選択されていること
- 未発行の仕入先が選択されていないこと
- WEBデポを使用していない仕入先が選択されていないこと

**処理後:**
- 支払確認書発行状況テーブルの公開フラグ = 1（仮公開）に更新
- WEBデポ担当者への仮公開通知メールを送信（API No.5）
- 明細の再検索を実行
