# Cline Workflow の活用シーンと実際の効果

## Workflow とは

Cline の **Workflow** は、繰り返し実行する複数ステップの作業を `.md` ファイルに定義しておき、`/workflow_name` のスラッシュコマンドで即座に呼び出せる機能です。

---

## 主な活用シーン

### 1. 定型作業の自動化（最も基本的なユースケース）

**シーン例：Backlog チケットの作業開始ルーティン**
```
チケット番号を入力するだけで：
 ① get_issue でチケット内容取得
 ② ステータスを「処理中」に変更
 ③ 作業開始コメントを自動投稿
 ④ 関連ドキュメントを自動読み込み
```

→ 毎回同じ手順を口頭で指示する必要がなくなる

---

### 2. コードレビュー・品質チェックの標準化

**シーン例：PR 作成前チェックワークフロー**
```
/pre_pr_check
 ① 変更ファイルの差分確認
 ② コーディング規約違反のチェック
 ③ テスト実行
 ④ PR テンプレートの自動生成
```

→ レビュアー・開発者ごとのチェック漏れを防止

---

### 3. ドキュメント作成・更新の一貫性確保

**シーン例：設計書作成ワークフロー（本プロジェクト向け）**
```
/create_design_doc [機能ID]
 ① 既存の関連設計書を読み込み
 ② テンプレートに沿って設計書を生成
 ③ docs/design/detail/ に保存
 ④ Backlog チケットにファイルパスをコメント
 ⑤ Backlog Wiki に同期
```

---

### 4. デプロイ・リリース作業の標準化

**シーン例：リリース準備ワークフロー**
```
/release_prep [バージョン]
 ① CHANGELOG 自動更新
 ② バージョン番号の一括書き換え
 ③ タグ作成・Push
 ④ リリースノートの Backlog Wiki 反映
```

---

### 5. 障害・インシデント対応の初動標準化

**シーン例：障害対応ワークフロー**
```
/incident_start [概要]
 ① 障害チケット自動作成（優先度：高）
 ② 対応ログ用 Wiki ページ作成
 ③ 関連コードの調査開始
 ④ 暫定対応メモの記録
```

---

## 実際の効果（メリット）

| 観点 | 効果 |
|---|---|
| **時間削減** | 定型指示の入力が不要になり、1タスクあたり 3〜10分の節約 |
| **ミス防止** | 手順の抜け漏れ・コンテキスト忘れを防止 |
| **品質均一化** | チームメンバー間で同じ手順・品質を担保 |
| **オンボーディング** | 新メンバーが複雑なルールを覚えなくても作業開始できる |
| **再現性** | 同じ作業を同じ品質で何度でも再現できる |

---

## 本プロジェクト（RXP）での応用可能性

`.clinerules` や `docs/rules/backlog_operations.md` に定義されている運用ルールは、**そのままWorkflowに変換**できます：

- `/backlog_start [チケットID]` → 作業開始の全手順を自動実行
- `/backlog_done [チケットID]` → 完了処理（ステータス変更・コメント・Wiki同期）を自動実行
- `/sync_wiki` → MDファイルとBacklog Wikiの同期を自動実行

---

## まとめ

Workflow は「**Clineへの指示書**」をコマンド化する機能です。
複数ステップにわたる定型作業を `.md` ファイル1枚で定義し、スラッシュコマンド1つで実行できるため、**チームの暗黙知をワークフローとして明文化・共有**する手段としても非常に有効です。

具体的なWorkflowファイルの作成・実装が必要であれば、Act Mode に切り替えて対応できます。
