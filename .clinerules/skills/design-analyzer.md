# 設計データ分析スキル (design-analyzer)

## 説明
解析後の設計データを分析し、設計パターン、潜在的な問題、最適化提案を識別し、設計判断記録（ADR）を生成します。アーキテクチャ設計レビューと技術方針策定に適しています。

## 入力
- `output/parse_result/` ディレクトリ内のMarkdown解析結果ファイル（`*_parse.md`）

## 出力
- `output/analysis_result/` ディレクトリに **Markdown形式** で分析レポートを保存
- ファイル名: `{元ファイル名}_analysis.md`

## 分析観点

### 1. アーキテクチャパターン分析
- MVC/レイヤードアーキテクチャの適合性
- Controller → Service → Repository の責務分離
- DTO/Entityの設計妥当性

### 2. Javaクラス設計チェック
- クラスの命名規則（Controller, Service, Repository, Dto等）
- メソッドの責務が単一責任原則に沿っているか
- 引数/戻り値の型が適切か

### 3. JavaScript/画面制御チェック
- 画面項目と制御ロジックの対応
- イベントハンドリングの設計
- バリデーションの配置（フロント/バック）

### 4. DB操作チェック
- テーブル操作のCRUD網羅性
- SQLインジェクション対策の設計
- トランザクション境界の明確性

### 5. 共通機能利用チェック
- フレームワーク共通機能の活用度
- 独自実装 vs 共通機能の判断妥当性

### 6. セキュリティチェック
- 認証/認可の設計
- 入力値検証の網羅性
- 機密情報の取り扱い

- 重要度は「高/中/低」の3段階で評価すること
- 改善提案は具体的かつ実行可能な内容にすること
