# Cline 功能设置说明

> **验证版本**
> - Visual Studio Code: `1.110`
> - Cline 扩展: `3.79.0`

## AGENT（智能体）部分

### Subagents（子智能体）
| 状态 | 行为 |
|------|------|
| **ON** | Cline 会并行启动多个子智能体，同时对代码库进行探索和分析。适合将复杂任务拆分后高效处理。 |
| **OFF** | 只有一个智能体按顺序处理任务。适合简单任务或需要精细控制执行过程的场景。 |

---

### Native Tool Call（原生工具调用）
| 状态 | 行为 |
|------|------|
| **ON** | 使用 AI 模型的原生函数调用功能（仅支持的模型）。可提升工具调用的精度和可靠性。 |
| **OFF** | 使用基于文本的工具调用方式。适合需要与某些特定模型或环境保持兼容性的场景。 |

---

### Parallel Tool Calling（并行工具调用）
| 状态 | 行为 |
|------|------|
| **ON** | 同时执行多个工具调用，大幅提升处理速度，实现高效任务处理。 |
| **OFF** | 逐个顺序执行工具调用。适合需要逐步确认每次工具调用结果的场景。 |

---

### Strict Plan Mode（严格计划模式）
| 状态 | 行为 |
|------|------|
| **ON** | 在 Plan 模式期间完全禁止文件编辑。计划阶段只进行读取和分析，防止文件被意外修改。 |
| **OFF** | 在 Plan 模式期间也可以编辑文件。适合希望灵活混合计划与执行操作的场景。 |

---

### Auto Compact（自动压缩）
| 状态 | 行为 |
|------|------|
| **ON** | 当对话历史接近上下文限制时，自动压缩/总结旧历史以节省上下文空间。即使长时间工作也能持续使用。 |
| **OFF** | 不自动压缩对话历史。达到上下文上限后需要手动处理。 |

---

### Focus Chain（焦点链）
| 状态 | 行为 |
|------|------|
| **ON** | 跨多轮交互保持上下文焦点。按照「Reminder Interval（1～10）」设置的间隔（默认：6）定期提醒模型回顾原始任务，防止长时间会话中偏离目标。 |
| **OFF** | 禁用上下文自动提醒功能。适合短任务或能自行管理上下文的场景。 |

---

## EDITOR（编辑器）部分

### Feature Tips（功能提示）
| 状态 | 行为 |
|------|------|
| **ON** | 在 Cline 思考阶段，循环展示 Cline 各项功能的使用提示，帮助用户自然地了解新功能和便捷用法。 |
| **OFF** | 不显示提示信息。适合已熟悉 Cline 的用户，或希望界面保持简洁的场景。 |

---

### Background Edit（后台编辑）
| 状态 | 行为 |
|------|------|
| **ON** | 在不抢占编辑器焦点的情况下在后台进行文件编辑。Cline 编辑文件期间，用户仍可自由浏览和编辑其他文件。 |
| **OFF** | 编辑文件时编辑器焦点会跳转到被编辑的文件。适合希望实时查看编辑内容的场景。 |

---

### Checkpoints（检查点）
| 状态 | 行为 |
|------|------|
| **ON** | 在任务的重要节点自动保存进度状态。出现问题时可以轻松回滚到任意检查点。重要工作建议开启。 |
| **OFF** | 不自动保存进度。适合不需要回滚功能，或已使用 Git 等其他方式进行版本管理的场景。 |

---

## EXPERIMENTAL（实验性功能）部分

> ⚠️ 这些功能仍处于实验阶段，请在充分理解后谨慎使用。

### Yolo Mode（极速模式）
| 状态 | 行为 |
|------|------|
| **ON** | 完全自动执行任务，无需用户确认。自动从 Plan 模式切换到 Act 模式，并禁用提问工具。**误操作和意外修改的风险极高，请极度谨慎使用。** |
| **OFF** | 每次操作均正常向用户确认。可保障操作安全性，通常建议保持关闭。 |

---

### Double-Check Completion（完成二次验证）
| 状态 | 行为 |
|------|------|
| **ON** | 拒绝首次完成尝试，要求模型将结果与原始任务要求重新核对后再接受完成。可提升成果质量和准确性。 |
| **OFF** | 直接接受首次完成尝试。适合优先速度或简单任务的场景。 |

---

### Lazy Teammate Mode（懒惰队友模式）
| 状态 | 行为 |
|------|------|
| **ON** | 模拟「今天 Cline 不在状态」。**纯娱乐功能**，没有实际用途 😄 |
| **OFF** | 作为正常的 Cline 运行。实际工作中建议始终关闭。 |

---

## ADVANCED（高级设置）部分

### Hooks（钩子）
| 状态 | 行为 |
|------|------|
| **ON** | 在任务执行过程中启用生命周期钩子和工具钩子。可以在特定事件（如工具调用前后）触发自定义处理逻辑，实现高度定制化。 |
| **OFF** | 禁用钩子功能。不需要自定义钩子，或希望保持简单行为时使用。 |

---

### MCP Display Mode（MCP 显示模式）
控制 MCP 服务器响应内容的显示方式，可从以下三个选项中选择：

| 选项 | 行为 |
|------|------|
| **Plain Text** | 以纯文本形式显示响应内容。简洁易读，为默认设置。 |
| **Rich Display** | 以富文本形式渲染响应内容。视觉上更整洁美观，便于阅读。 |
| **Markdown** | 将响应内容解析为 Markdown 格式并渲染。可显示标题、列表等带格式的内容。 |

---

## 推荐设置总结

| 使用场景 | 推荐设置 |
|----------|----------|
| **日常使用** | Native Tool Call・Parallel Tool Calling・Checkpoints・Hooks 开启 |
| **注重质量** | 开启 Double-Check Completion |
| **长时间工作** | 开启 Auto Compact・Focus Chain |
| **注重安全** | 开启 Strict Plan Mode，关闭 Yolo Mode |
| **追求自动化** | Yolo Mode 虽然方便，但误操作风险高，请谨慎使用 |
