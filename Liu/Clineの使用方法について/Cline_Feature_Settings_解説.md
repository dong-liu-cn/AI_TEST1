# Cline Feature Settings 機能設定 解説

> **動作確認バージョン**
> - Visual Studio Code: `1.110`
> - Cline 拡張機能: `3.79.0`

## AGENT（エージェント）セクション

### Subagents（サブエージェント）
| 状態 | 動作 |
|------|------|
| **ON** | Clineが複数のサブエージェントを並列で起動し、コードベースの探索や分析を同時に行う。複雑なタスクを分割して効率的に処理できる。 |
| **OFF** | 1つのエージェントのみが順番にタスクを処理する。シンプルなタスクや制御を細かく行いたい場合に適している。 |

---

### Native Tool Call（ネイティブツール呼び出し）
| 状態 | 動作 |
|------|------|
| **ON** | 対応しているAIモデルのネイティブ関数呼び出し機能を使用する。ツール呼び出しの精度と信頼性が向上する。 |
| **OFF** | テキストベースのツール呼び出し方式を使用する。一部のモデルや環境との互換性が必要な場合に使用。 |

---

### Parallel Tool Calling（並列ツール呼び出し）
| 状態 | 動作 |
|------|------|
| **ON** | 複数のツール呼び出しを同時に実行する。処理速度が大幅に向上し、効率的なタスク処理が可能になる。 |
| **OFF** | ツール呼び出しを1つずつ順番に実行する。各ツール呼び出しの結果を確認しながら進めたい場合に有効。 |

---

### Strict Plan Mode（厳格なプランモード）
| 状態 | 動作 |
|------|------|
| **ON** | Planモード中はファイルの編集を一切禁止する。計画フェーズでは読み取りと分析のみ行い、誤ってファイルが変更されるのを防ぐ。 |
| **OFF** | Planモード中もファイルの編集が可能になる。柔軟に計画と実行を混在させたい場合に使用。 |

---

### Auto Compact（自動圧縮）
| 状態 | 動作 |
|------|------|
| **ON** | 会話履歴がコンテキスト上限に近づいたとき、自動的に古い履歴を圧縮・要約してコンテキスト枠を節約する。長時間の作業でも継続して使用できる。 |
| **OFF** | 会話履歴の自動圧縮を行わない。コンテキスト上限に達した場合は手動で対処が必要になる。 |

---

### Focus Chain（フォーカスチェーン）
| 状態 | 動作 |
|------|------|
| **ON** | 複数のやり取りをまたいでコンテキストの焦点を維持する。「Reminder Interval（1〜10）」で設定した間隔（デフォルト：6）ごとに、元のタスクを念頭に置くよう促す。長いセッションでの目的の逸脱を防ぐ。 |
| **OFF** | コンテキストの自動リマインド機能が無効になる。短いタスクや自分でコンテキストを管理できる場合に適している。 |

---

## EDITOR（エディター）セクション

### Feature Tips（機能ヒント）
| 状態 | 動作 |
|------|------|
| **ON** | Clineの思考フェーズ中に、Clineの様々な機能に関するヒントをローテーション表示する。新機能や便利な使い方を自然に学べる。 |
| **OFF** | ヒントの表示を行わない。Clineに慣れたユーザーや、表示をシンプルに保ちたい場合に適している。 |

---

### Background Edit（バックグラウンド編集）
| 状態 | 動作 |
|------|------|
| **ON** | エディターのフォーカスを奪わずにバックグラウンドでファイル編集を行う。Clineが編集中でも他のファイルを自由に閲覧・編集できる。 |
| **OFF** | ファイル編集時にエディターのフォーカスが編集対象ファイルに移動する。編集内容をリアルタイムで確認したい場合に適している。 |

---

### Checkpoints（チェックポイント）
| 状態 | 動作 |
|------|------|
| **ON** | タスクの重要な節目ごとに進捗状態を自動保存する。問題が発生した場合、任意のチェックポイントに簡単にロールバックできる。重要な作業では有効推奨。 |
| **OFF** | 進捗の自動保存を行わない。ロールバック機能が不要な場合や、Gitなど別の方法でバージョン管理している場合に使用。 |

---

## EXPERIMENTAL（実験的機能）セクション

> ⚠️ これらの機能は実験段階です。十分に理解した上で使用してください。

### Yolo Mode（ヨーロモード）
| 状態 | 動作 |
|------|------|
| **ON** | ユーザーの確認なしにタスクを完全自動実行する。PlanモードからActモードへ自動切り替えし、質問ツールも無効化される。**誤操作・意図しない変更のリスクが非常に高いため、極めて慎重に使用すること。** |
| **OFF** | 通常通り各操作でユーザーの確認を求める。安全な操作が保証されるため、通常はOFFを推奨。 |

---

### Double-Check Completion（完了二重チェック）
| 状態 | 動作 |
|------|------|
| **ON** | タスク完了の最初の試みを一度却下し、元のタスク要件と照らし合わせてモデルに再検証させてから完了とする。成果物の品質と正確性が向上する。 |
| **OFF** | 最初の完了試行をそのまま受理する。処理速度を優先する場合や、シンプルなタスクに適している。 |

---

### Lazy Teammate Mode（怠惰なチームメートモード）
| 状態 | 動作 |
|------|------|
| **ON** | 「今日はClineがやる気ない日」をシミュレートする。**エンターテイメント目的のみの冗談機能**であり、実用的な用途はない 😄 |
| **OFF** | 通常のClineとして動作する。実際の業務では常にOFFを推奨。 |

---

## ADVANCED（詳細設定）セクション

### Hooks（フック）
| 状態 | 動作 |
|------|------|
| **ON** | タスク実行中にライフサイクルフックやツールフックを有効にする。特定のイベント（ツール呼び出し前後など）に応じたカスタム処理を実行できる。高度なカスタマイズが可能。 |
| **OFF** | フック機能を無効にする。カスタムフックが不要な場合や、シンプルな動作を求める場合に使用。 |

---

### MCP Display Mode（MCPの表示モード）
MCPサーバーからのレスポンスをどのように表示するかを制御する。以下の3つから選択できる。

| 選択肢 | 動作 |
|--------|------|
| **Plain Text** | レスポンスをプレーンテキストとして表示する。シンプルで読みやすく、デフォルト設定。 |
| **Rich Display** | リッチな表示形式でレスポンスをレンダリングする。視覚的に整理された見やすい表示が可能。 |
| **Markdown** | レスポンスをMarkdown形式として解釈・レンダリングする。見出しや箇条書きなど書式付きで表示できる。 |

---

## おすすめ設定まとめ

| 用途 | おすすめ設定 |
|------|------------|
| **通常使用** | Native Tool Call・Parallel Tool Calling・Checkpoints・Hooks をON |
| **品質重視** | Double-Check Completion をON |
| **長時間作業** | Auto Compact・Focus Chain をON |
| **安全重視** | Strict Plan Mode をON、Yolo Mode はOFF |
| **自動化重視** | Yolo Mode は便利だが誤操作リスクが高いため慎重に使用 |
