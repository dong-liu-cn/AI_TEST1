# Cline 与 Cline CLI 区别详情调研报告

## 概述

Cline 是一个开源的 AI 编程助手，最初作为 VS Code 扩展发布，后来推出了命令行界面版本 Cline CLI。两者共享相同的核心功能，但在使用场景、交互方式和功能特性上存在显著差异。

## 核心区别对比

| 特性维度 | Cline (VS Code 扩展) | Cline CLI (命令行工具) |
|---------|-------------------|---------------------|
| **产品形态** | VS Code 扩展插件 | 命令行工具 (Node.js 包) |
| **安装方式** | VS Code 扩展市场安装 | `npm install -g cline` |
| **运行环境** | 必须在 VS Code 内运行 | 可在任何终端中运行 |
| **交互界面** | 图形化界面 (GUI) | 终端用户界面 (TUI) / 纯文本 |
| **主要使用场景** | 集成开发环境内的编码辅助 | 自动化脚本、CI/CD、终端工作流 |
| **并行处理能力** | 单实例运行 | 支持多实例并行运行 |
| **集成方式** | 深度集成 VS Code API | 通过 ACP (Agent Client Protocol) 连接各种编辑器 |
| **自动化程度** | 交互式为主 | 支持无头 (headless) 自动化模式 |

## 详细功能对比

### 1. Cline (VS Code 扩展)

**核心特性：**
- **IDE 深度集成**：作为 VS Code 扩展，直接嵌入编辑器界面
- **图形化交互**：提供侧边栏聊天面板，可视化操作界面
- **文件操作**：直接在编辑器中创建、编辑、删除文件
- **代码上下文**：通过 `@file`、`@folder`、`@url` 等指令引用项目文件
- **终端集成**：在 VS Code 终端中执行命令并监控输出
- **浏览器支持**：启动无头浏览器进行网页测试和调试
- **MCP 支持**：通过 Model Context Protocol 连接外部工具
- **Plan/Act 模式**：分离计划制定和代码执行阶段

**使用流程：**
1. 在 VS Code 扩展市场安装 Cline
2. 配置 API 提供商和模型
3. 通过侧边栏图标打开 Cline 面板
4. 输入任务描述，Cline 会分析并执行

**优势：**
- 与开发环境无缝集成
- 可视化操作，适合交互式开发
- 直接访问编辑器功能（如代码高亮、错误诊断）
- 适合日常编码和调试任务

### 2. Cline CLI

**核心特性：**
- **终端原生**：直接在终端中运行，无需图形界面
- **两种运行模式**：
  - **交互模式**：启动 TUI 界面，类似 VS Code 扩展的体验
  - **无头模式**：通过 `-y` 标志实现完全自动化执行
- **并行实例**：可同时运行多个独立实例，处理不同任务
- **管道集成**：支持标准输入/输出，可与其他命令行工具组合
- **脚本友好**：支持 JSON 输出格式，便于脚本处理
- **ACP 协议**：通过 Agent Client Protocol 连接到各种编辑器

**安装要求：**
- Node.js 20 或更高版本（推荐 Node.js 22）
- 通过 `npm install -g cline` 全局安装
- 使用 `cline auth` 进行身份验证和模型配置

**优势：**
- 适合自动化工作流和 CI/CD 管道
- 支持并行处理，提高效率
- 与现有 shell 工具链集成
- 可在无图形界面的服务器环境中运行

## 适用场景对比

### 适合使用 Cline (VS Code 扩展) 的场景：
- 日常编码和调试工作
- 需要与编辑器深度集成的任务
- 可视化代码审查和修改
- 交互式项目探索和重构
- 需要直接操作编辑器功能的情况

### 适合使用 Cline CLI 的场景：
- 自动化代码维护和重构
- CI/CD 管道集成
- 批量处理任务
- 服务器环境部署
- 与其他命令行工具集成的工作流
- 需要并行处理多个任务的情况
- 终端优先的开发工作流

## 总结

Cline 和 Cline CLI 是同一产品的两种不同形态，服务于不同的使用场景：

- **Cline (VS Code 扩展)** 更适合**交互式开发**，提供图形化界面和深度 IDE 集成，适合日常编码、调试和项目探索。

- **Cline CLI** 更适合**自动化和脚本化工作流**，提供终端原生体验、并行处理和管道集成，适合 CI/CD、批量任务和服务器环境。
