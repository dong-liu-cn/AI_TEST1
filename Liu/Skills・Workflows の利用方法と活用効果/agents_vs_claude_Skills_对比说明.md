# `.agents` 技能 vs `.claude/skills` 技能 — 区别与使用指南

---

## 1. 概述

在使用 Cline 等 AI 智能体开发时，"技能（skill）"可以定义在项目中的两个主要位置：

| 存放位置 | 适用框架 |
|----------|----------|
| `.agents/` | 通用多智能体框架（OpenAI Agents SDK 等） |
| `.claude/skills/` | 仅适用于 Cline（VS Code 扩展插件） |

这两者在**目的、机制、描述形式**上完全不同，理解各自的特性至关重要，避免混淆使用。

---

## 2. `.agents/` 技能是什么

### 2.1 定位

`.agents/` 目录用于**多智能体框架**（OpenAI Agents SDK、LangGraph、AutoGen 等），存放智能体定义、工具注册和技能实现代码。

### 2.2 工作机制

- 智能体运行时启动时自动扫描该目录，**自动检测并注册**技能。
- 技能以**程序代码**（Python / TypeScript 的函数或类）形式编写。
- 多个智能体之间可以**共享和继承**技能。

### 2.3 编写形式

```python
# .agents/search_skill.py 示例（OpenAI Agents SDK）
from agents import function_tool

@function_tool
def search_web(query: str) -> str:
    """搜索网络并返回结果"""
    # 实现逻辑...
    return result
```

### 2.4 特点

| 项目 | 内容 |
|------|------|
| **编写语言** | Python / TypeScript 等编程语言 |
| **运行环境** | 服务器、CI/CD、后端运行时 |
| **触发方式** | 代码内调用智能体，或自动注册为工具 |
| **作用范围** | 整个程序运行环境 |
| **可复用性** | 可在多个智能体间共享和继承 |
| **版本管理** | 作为代码文件使用 Git 管理 |

### 2.5 主要用途

- 为**自主智能体**赋予特定能力（Web 搜索、数据库操作、API 调用）
- 在 **CI/CD 流水线**中定义自动化处理技能
- 在多智能体架构中实现**按角色分离技能集**

---

## 3. `.claude/skills/` 技能是什么

### 3.1 定位

`.claude/skills/` 目录是 **Cline（VS Code 扩展插件）专用**的技能定义位置。将其放置在项目根目录后，Cline 在该项目中工作时会自动识别。

### 3.2 工作机制

- Cline 启动时读取 `.claude/skills/` 中的 Markdown 文件，自动列举到**系统提示词的 `SKILLS` 章节**中。
- 用户或 Cline 自身调用 `use_skill` 工具并指定技能名称后，技能被**激活**，Cline 按照该技能的指示运作。
- 技能以 **Markdown 文件**（自然语言指示书）形式编写。

### 3.3 编写形式

```markdown
<!-- .claude/skills/design-analyzer.md 示例 -->
# design-analyzer

## 说明
分析设计数据，识别设计模式与潜在问题。

## 步骤
1. 读取输入的设计文档文件
2. 提取类定义与方法定义
3. 识别设计模式
4. 输出问题点与改进建议的报告
```

### 3.4 特点

| 项目 | 内容 |
|------|------|
| **编写语言** | Markdown（自然语言指示书） |
| **运行环境** | VS Code 的 Cline 聊天界面内 |
| **触发方式** | 通过 `use_skill` 工具指定技能名称 |
| **作用范围** | 仅在当前 Cline 会话内有效 |
| **可复用性** | 同一项目中所有 Cline 会话均可使用 |
| **版本管理** | 作为 Markdown 文件使用 Git 管理 |

### 3.5 主要用途

- 将**固定流程的操作手册**存入 Cline，供反复调用
- 将项目特有的**编码规范和设计模式**应用到 Cline 的工作中
- 将**复杂的多步骤任务**（如 Excel 解析 → 设计文档生成）一键执行

## 4. 对比表

| 对比项目 | `.agents/` 技能 | `.claude/skills/` 技能 |
|----------|-----------------|------------------------|
| **适用框架** | OpenAI Agents SDK / LangGraph / AutoGen 等 | 仅 Cline（VS Code 扩展插件） |
| **存放位置** | 项目根目录下的 `.agents/` | 项目根目录下的 `.claude/skills/` |
| **编写形式** | Python / TypeScript 代码 | Markdown（自然语言） |
| **加载方式** | 智能体运行时自动检测并注册 | Cline 自动列举到系统提示词中 |
| **触发方式** | 代码内调用智能体 | `use_skill` 工具指定技能名称 |
| **运行环境** | 服务器 / CI/CD / 后端 | VS Code 的 Cline 聊天界面 |
| **作用范围** | 整个程序运行环境 | 仅当前 Cline 会话 |
| **智能体间共享** | ✅ 可以（支持继承与共享） | ❌ 不可（仅 Cline 单体） |
| **编写难度** | 高（需要编程知识） | 低（用自然语言即可） |
| **版本管理** | Git（代码文件） | Git（Markdown 文件） |
| **主要用途** | 为自主智能体赋予能力 | 定义 Cline 的操作流程与规则 |

## 5. 使用选择指南

### 应选择 `.agents/` 技能的情况

- 正在构建或运营**多智能体系统**
- 需要在 CI/CD 或**服务器端自动执行**的处理中定义技能
- 多个智能体需要**共享同一技能**
- 技能中包含**复杂逻辑**（API 调用、数据转换、计算处理等）
- 由具备 Python / TypeScript 实现能力的工程师负责

### 应选择 `.claude/skills/` 技能的情况

- 希望提升**使用 Cline 进行日常开发工作**的效率
- 希望让 Cline 记住特定项目的**编码规范或设计流程**
- 希望将**反复执行的固定任务**（设计文档解析、代码审查、文档生成等）一键化
- 希望**不编程，用自然语言定义技能**

### 两者结合使用的情况

在大型项目中，两者可以同时并用：

```
项目根目录/
├── .agents/
│   ├── api_skill.py        # 后端自主智能体技能
│   └── db_skill.py         # 数据库操作智能体技能
└── .claude/
    └── skills/
        ├── code-reviewer.md    # Cline 代码审查流程
        └── design-analyzer.md  # Cline 设计文档解析流程
```

这种结构实现了清晰的职责分工：
- `.agents/` → **生产环境 / CI/CD 的智能体处理**
- `.claude/skills/` → **开发者的 Cline 工作辅助**

---

## 6. 总结

| | `.agents/` | `.claude/skills/` |
|--|------------|-------------------|
| **一句话概括** | 面向多智能体框架的**代码技能** | Cline 专用的**操作手册技能** |
| **谁来使用** | 智能体运行时（自动） | 开发者通过 Cline（手动/自动） |
| **写什么内容** | 可执行的程序逻辑 | 自然语言的操作指示 |

简单来说：`.agents/` 是系统架构层面的技能管理，`.claude/skills/` 是面向个人开发者提升工作效率的技能管理。理解这一本质区别，就能做出正确的使用选择。
