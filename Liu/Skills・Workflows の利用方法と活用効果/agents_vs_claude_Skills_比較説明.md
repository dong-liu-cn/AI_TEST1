# `.agents` スキル vs `.claude/skills` スキル — 違いと使い分けガイド

---

## 1. 概要

Cline を含む AI エージェント開発では、「スキル（skill）」をプロジェクト内に定義する場所として主に以下の2つが存在します。

| 配置場所 | 対象フレームワーク |
|----------|-------------------|
| `.agents/` | マルチエージェントフレームワーク全般（OpenAI Agents SDK など） |
| `.claude/skills/` | Cline（VS Code 拡張機能）専用 |

これらは**目的・仕組み・記述形式**がまったく異なります。混同しないよう、それぞれの特性を理解することが重要です。

---

## 2. `.agents/` のスキルとは

### 2.1 位置づけ

`.agents/` ディレクトリは、**マルチエージェントフレームワーク**（OpenAI Agents SDK、LangGraph、AutoGen など）において、エージェントの定義・ツール登録・スキル実装を格納する場所です。

### 2.2 仕組み

- エージェントランタイムが起動時にディレクトリをスキャンし、スキルを**自動検出・登録**します。
- スキルは**プログラムコード**（Python / TypeScript の関数・クラス）として記述します。
- エージェント同士でスキルを**共有・継承**することが可能です。

### 2.3 記述形式

```python
# .agents/search_skill.py の例（OpenAI Agents SDK）
from agents import function_tool

@function_tool
def search_web(query: str) -> str:
    """Web を検索して結果を返す"""
    # 実装...
    return result
```

### 2.4 特徴

| 項目 | 内容 |
|------|------|
| **記述言語** | Python / TypeScript などのプログラミング言語 |
| **実行環境** | サーバー・CI/CD・バックエンドのランタイム |
| **発動方法** | コード内でのエージェント呼び出し、または自動ツール登録 |
| **スコープ** | プログラム実行環境全体 |
| **再利用性** | 複数エージェント間でスキルを共有・継承可能 |
| **バージョン管理** | コードとして Git 管理 |

### 2.5 主な用途

- **自律型エージェント**に特定の能力（Web 検索、DB 操作、API 呼び出し）を付与する
- **CI/CD パイプライン**内での自動処理スキルの定義
- マルチエージェント構成における**ロール別スキルセットの分離**

---

## 3. `.claude/skills/` のスキルとは

### 3.1 位置づけ

`.claude/skills/` ディレクトリは、**Cline（VS Code 拡張機能）専用**のスキル定義場所です。プロジェクトルートに配置することで、Cline がそのプロジェクトで作業する際に自動的に認識します。

### 3.2 仕組み

- Cline が起動時に `.claude/skills/` 内の Markdown ファイルを読み込み、**システムプロンプトの `SKILLS` セクション**に自動列挙します。
- ユーザーまたは Cline 自身が `use_skill` ツールを呼び出すことでスキルが**アクティベート**され、そのスキルの指示に従って動作します。
- スキルは**Markdown ファイル**（自然言語の指示書）として記述します。

### 3.3 記述形式

```markdown
<!-- .claude/skills/design-analyzer.md の例 -->
# design-analyzer

## 説明
設計データを分析し、設計パターンと潜在的な問題を識別します。

## 手順
1. 入力された設計書ファイルを読み込む
2. クラス定義・メソッド定義を抽出する
3. 設計パターンを識別する
4. 問題点と改善提案をリポートとして出力する
```

### 3.4 特徴

| 項目 | 内容 |
|------|------|
| **記述言語** | Markdown（自然言語の指示書） |
| **実行環境** | VS Code の Cline チャット内 |
| **発動方法** | `use_skill` ツールでスキル名を指定 |
| **スコープ** | Cline セッション内のみ有効 |
| **再利用性** | 同じプロジェクト内の全 Cline セッションで利用可能 |
| **バージョン管理** | Markdown ファイルとして Git 管理 |

### 3.5 主な用途

- **定型作業の手順書**を Cline に記憶させ、繰り返し使用する
- プロジェクト固有の**コーディング規約・設計パターン**を Cline に適用する
- **複雑な多段階タスク**（Excel 解析 → 設計書生成など）を1コマンドで実行させる

## 4. 比較表

| 比較項目 | `.agents/` スキル | `.claude/skills/` スキル |
|----------|-------------------|--------------------------|
| **対象フレームワーク** | OpenAI Agents SDK / LangGraph / AutoGen など | Cline（VS Code 拡張機能）のみ |
| **配置場所** | プロジェクトルート直下の `.agents/` | プロジェクトルート直下の `.claude/skills/` |
| **記述形式** | Python / TypeScript コード | Markdown（自然言語） |
| **読み込み方法** | エージェントランタイムが自動検出・登録 | Cline がシステムプロンプトに自動列挙 |
| **発動方法** | コード内でのエージェント呼び出し | `use_skill` ツールでスキル名を指定 |
| **実行環境** | サーバー・CI/CD・バックエンド | VS Code の Cline チャット内 |
| **スコープ** | プログラム実行環境全体 | Cline セッション内のみ |
| **エージェント間共有** | ✅ 可能（継承・共有） | ❌ 不可（Cline 単体のみ） |
| **記述の難易度** | 高（プログラミング知識が必要） | 低（自然言語で記述可能） |
| **バージョン管理** | Git（コードファイル） | Git（Markdown ファイル） |
| **主な用途** | 自律エージェントへの能力付与 | Cline の作業手順・ルールの定義 |

## 5. 使い分けのガイドライン

### `.agents/` スキルを選ぶべき場合

- **マルチエージェントシステム**を構築・運用している
- CI/CD や**サーバーサイドで自動実行**される処理にスキルが必要
- 複数のエージェントで**同一スキルを共有**したい
- スキルに**複雑なロジック**（API 呼び出し、データ変換、計算処理）が含まれる
- Python / TypeScript での実装が可能なエンジニアが担当する

### `.claude/skills/` スキルを選ぶべき場合

- **Cline を使った日常の開発作業**を効率化したい
- 特定プロジェクトの**コーディング規約や設計手順**を Cline に覚えさせたい
- **繰り返し実行する定型タスク**（設計書解析、コードレビュー、ドキュメント生成など）をワンコマンド化したい
- プログラミングなしに**自然言語でスキルを定義**したい

### 両方を組み合わせる場合

大規模プロジェクトでは両方を併用することもあります：

```
プロジェクトルート/
├── .agents/
│   ├── api_skill.py        # バックエンドの自律エージェント用スキル
│   └── db_skill.py         # DB 操作エージェント用スキル
└── .claude/
    └── skills/
        ├── code-reviewer.md    # Cline によるコードレビュー手順
        └── design-analyzer.md  # Cline による設計書解析手順
```

この構成では：
- `.agents/` → **本番環境・CI/CD でのエージェント処理**
- `.claude/skills/` → **開発者の Cline 作業支援**

という役割分担が明確になります。

## 6. まとめ

| | `.agents/` | `.claude/skills/` |
|--|------------|-------------------|
| **一言で言うと** | マルチエージェントフレームワーク向けの**コードスキル** | Cline 専用の**手順書スキル** |
| **誰が使うか** | エージェントランタイム（自動） | 開発者が Cline を通じて（手動/自動） |
| **何を記述するか** | 実行可能なプログラムロジック | 自然言語による作業指示 |

`.agents/` はシステム・アーキテクチャレベルのスキル管理、`.claude/skills/` は個々の開発者の作業効率化のためのスキル管理と理解すると、適切な使い分けができます。
