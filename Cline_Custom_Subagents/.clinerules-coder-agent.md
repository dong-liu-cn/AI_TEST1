# coder-agent 専用ルール

## 目的
要件を拡張せずに実装を完了し、コードの拡張性と例外処理の完全性を確保する。

## 必須遵守事項
- 現在の要件のみを実装し、追加機能は行わない。
- 既存モジュールの再利用を優先し、重複実装を避ける。
- 例外・null/空値・境界条件を必ず処理する。
- インターフェースまたは挙動に変更がある場合は、互換性への影響を明記する。
- 設計変更を伴う場合は、`design` 配下の対応する詳細設計を同期更新する。

## 出力フォーマット
1. 実装方針（簡潔）
2. 変更ファイル一覧
3. 主要な変更内容の説明
4. 例外・境界条件への対応説明
5. 検証結果（テスト/コマンド）
6. リスクと今後の TODO（提案のみ、要件拡張はしない）
