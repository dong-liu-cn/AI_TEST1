# Excel業務機能設計書解析スキル

## 概要
WMS（倉庫管理システム）等の業務機能設計書（Excel）を解析し、Markdown形式の基本設計書として出力するスキルです。

## 使い方

### 統合スクリプト（推奨）

```bash
py .agents/skills/excel-parse/scripts/excel_to_md.py --input "<入力Excelパス>" --output "<出力MDパス>"
```

**例：**
```bash
py .agents/skills/excel-parse/scripts/excel_to_md.py --input "skill/D-IM-010-P_業務機能設計書_在庫一覧(Webデポ)_Ver0.1.xlsx" --output "skill/D-IM-010-P_在庫一覧_基本設計書.md"
```

### 図形抽出のみ

```bash
py .agents/skills/excel-parse/scripts/extract_excel_shapes.py "<Excelパス>" "<シート名>"
```

## 自動シート分類

スクリプトはシート名のキーワードから自動的に以下のカテゴリに分類し、適切なパーサーを適用します：

| カテゴリ | キーワード例 | 処理内容 |
|---------|------------|---------|
| `change_history` | 変更履歴 | 表形式で変更履歴を抽出 |
| `spec_overview` | 仕様概要 | ヘッダ情報＋本文テキスト抽出 |
| `design_overview` | 設計概要 | テキスト段落として抽出 |
| `screen_items` | 画面項目定義 | 項目一覧テーブル抽出 |
| `screen_state` | 画面状態定義 | 状態遷移テーブル抽出 |
| `screen_check` | 画面項目チェック | チェック定義テーブル抽出 |
| `screen_source` | 画面取得元 | 取得元定義テーブル抽出 |
| `screen_transition` | 画面遷移図, 機能遷移図 | **図形抽出→Mermaid生成** |
| `process_detail` | 処理詳細 | 処理フローテーブル抽出 |
| `table_mapping` | テーブル項目マッピング | DB項目マッピングテーブル抽出 |
| `excel_transfer` | Excel移送表 | 移送定義テキスト抽出 |
| `excel_layout` | Excelレイアウト | レイアウト定義テキスト抽出 |

## 画面遷移図の処理（図形→Mermaid変換）

Excelの画面遷移図シートに含まれる図形（矢印・四角形）を解析し、以下を自動生成します：

### 出力例

```markdown
### 画面一覧（図形から抽出）

| No | 画面名 | 図形タイプ |
|---|---|---|
| 1 | 在庫一覧（Webデポ） | rect |
| 2 | パレットメニュー | rect |

### 遷移図

​```mermaid
graph LR
    A[パレットメニュー] -->|メニューボタン押下| B[在庫一覧（Webデポ）]
    B[在庫一覧（Webデポ）] -->|「詳細」ボタン押下| C[在庫詳細（Webデポ）]
​```

### 遷移定義テーブル

| No | 遷移元画面 | トリガー | 遷移先画面 | 遷移タイプ |
|----|-----------|---------|-----------|-----------|
| 1 | パレットメニュー | メニューボタン押下 | 在庫一覧（Webデポ） | 画面遷移 |
```

### 技術的な仕組み

1. `openpyxl` でExcelの `drawing` オブジェクトを取得
2. `TwoCellAnchor` から図形の位置・テキスト・接続線を解析
3. 矢印の `from/to` 座標と四角形の位置から接続関係を推定
4. Mermaid `graph LR` 記法に変換

### 制約事項

- **画像として貼り付けられたフローチャート**は解析不可（openpyxlの制限）
- Excel図形（Shape/Connector）として描かれたもののみ対応
- 図形内にテキストがない場合は「不明な画面」として出力
- 接続線のテキスト（トリガー）が取得できない場合は近傍テキストから推定

## 解析対象データ

### 1. 基本情報
- **業務ID/機能ID**：業務識別子、機能識別子
- **業務名称/機能名称**：業務名、機能名
- **作成者/審査者/承認者**：担当者情報
- **変更履歴**：変更年月日、変更シート、変更内容、種別

### 2. 処理詳細設計
- **処理一覧**：処理番号、処理名称、トリガー
- **クラス/メソッド定義**：クラスID、メソッドID
- **処理フロー**：処理No、処理種別、処理内容

### 3. 画面項目定義
- **ヘッダ部/明細部/フッタ部項目**：項目名、入力種別、取得元

### 4. テーブル項目マッピング
- **データ取得/登録/更新マッピング**：画面項目 ↔ テーブル ↔ カラム

### 5. 画面遷移図
- **図形抽出** → **Mermaid遷移図**自動生成
- **遷移定義テーブル**：遷移元・トリガー・遷移先・遷移タイプ

### 6. Excel帳票
- **Excel移送表**：項目マッピング定義
- **Excelレイアウト**：帳票レイアウト定義

## 依存ライブラリ

```
openpyxl
lxml
```

インストール：
```bash
pip install openpyxl lxml
```

## 注意事項
- Excel内のセル結合がある場合は、結合セルの値を上位セルから継承する
- クラスIDのファイル拡張子（`.js`/`.java`）からフロントエンド/バックエンドを判別する
- テーブル名は物理名（英字）と論理名（日本語）の両方を抽出する
- 画面遷移図は**Excel図形（Shape）**のみ対応、**画像貼り付け**には非対応
