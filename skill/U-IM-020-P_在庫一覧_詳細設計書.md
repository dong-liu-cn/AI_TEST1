### 3.2 アクション別処理詳細

#### ① 検索処理（「検索」ボタン押下）

**処理概要**：検索条件に従い、在庫情報を取得する。

**処理前提**：無し

**処理フロー**：

1. **入力チェック**（`StockListLogic.validateSearchCondition`）
   - 必須チェック：実績済在庫基準日、予定在庫基準日
   - 実績済在庫基準日 > 業務日付 → エラー（E.P20REN.CR.00989）
   - 予定在庫基準日 < 業務日付 → エラー（E.P20REN.CR.00987）
   - 入力タイプチェック（共通仕様書準拠）

2. **検索実行**（`StockListSelectLogic.selectStockList`）
   - 検索条件はすべてAND結合
   - **例外**：品目コード（複数）1～4はOR結合
   - 品目コードが入力されている場合、品目コード（複数）は検索条件に含めない
   - 取引先名（検索条件）は部分一致で検索

3. **集計処理**
   - 実績済在庫数 = 全ランクの基準日在庫 ＋ 未承認入庫実績 － 未承認出庫実績
   - 出庫可能数 = 実績済在庫数のうち「一般品」「限定品」の和
   - 予定在庫数 = 実績済在庫数 + 未実績オーダー入庫数 - 未実績オーダー出庫数 + 承認済在庫未反映実績数 + 仮オーダー入庫数 - 仮オーダー出庫数

4. **結果件数チェック**
   - 0件の場合 → エラー（E.P20REN.CR.00034）

5. **ソート処理**
   - 組織コード（優先表示組織を先頭） → 取引先コード → 品目コード
   - 優先表示組織（コードマスタから取得、例：1300、1303）を一覧先頭に表示し「在庫優先表示」に"○"を付与

**補足**：
- 検索処理の詳細は「設計概要①_概要補足」シートの＜１．検索結果について＞を参照
- 実績済在庫基準日と予定在庫基準日に従う在庫数の表示は「設計概要②_在庫の見え方」を参照

#### ② Excel出力処理（「Excel」ボタン押下）

**処理概要**：検索結果一覧をExcelで出力する。

**処理前提**：検索結果が一覧表示されていること。

**処理フロー**：

1. 必須チェック、入力タイプチェック実施
2. 検索結果データを取得
3. Excelファイル生成（共通Excel出力コンポーネント使用）
4. ダウンロード実行
5. 出力処理完了の確認メッセージを表示

**出力順**：「検索」ボタン押下時と同じ

**補足**：詳細は別紙の共通設計書を参照。

#### ③ ページング制御処理

**処理概要**：検索結果一覧のページング制御を行う。

**処理前提**：検索結果が一覧表示されていること。

**トリガー**：「先頭頁」「前頁」「後頁」「最終頁」ボタン

**補足**：詳細は別紙の共通設計書を参照。

#### ④ 在庫詳細確認処理（「選択」ボタン押下）

**処理概要**：該当レコードの在庫詳細画面へ参照画面として遷移する。

**遷移先**：在庫詳細画面（参照モード）

#### ⑤ 在庫振替入力処理（「振替」ボタン押下）

**処理概要**：該当レコードの在庫振替入力画面へ新規登録画面として遷移する。

**遷移先**：在庫振替入力画面（新規登録モード）

#### ⑥ 取引先コードリンク処理

**処理概要**：本画面の検索モードが「内訳」として、新規Windowで在庫一覧画面を表示する。

#### ⑦ 品目コードリンク処理

**処理概要**：本画面の検索モードが「内訳」として、新規Windowで在庫一覧画面を表示する。

#### ⑧ 在庫日報Excel出力処理

**処理概要**：Webデポで出力する在庫日報Excelの検索・出力を行い、結果をダウンロードする。

**処理前提**：
- 取引先コード・選別数集計基準日の必須チェック
- 選別数集計基準日の未来日チェック（営業日と比較）

**処理フロー**：

1. 必須チェック（取引先コード、選別数集計基準日）
2. 未来日チェック（選別数集計基準日 ≤ 営業日）
3. 在庫日報データ検索（会社コード="100"、ロケール="ja"固定）
4. 検索基準日（実績済在庫基準日）以前で最も近い在庫日報を出力
5. Excelファイル生成（日本語テンプレート使用）
6. ダウンロード実行
7. 完了メッセージ表示：「在庫日報の発行が完了しました。」

## 第9章 業務ルール

### 9.1 検索ロジック

- 各検索条件は**AND**で結合する
- **例外**：品目コード（複数）1～4間のみ**OR**で結合する
- 品目コードが入力されている場合、品目コード（複数）は検索条件に含めず品目コードで検索する
- 取引先名（検索条件）は**部分一致**で検索する

### 9.2 優先表示ルール

- コードマスタに設定されている優先表示組織（例：1300、1303）を一覧の先頭に表示する
- 該当レコードの「在庫優先表示」に"○"を付与する

### 9.3 在庫日報固定値

- 会社コード：100（固定）
- ロケール：ja（固定）
- ログインユーザの所属会社・ロケールにかかわらず、Webデポ同様の設定で各種名称を取得する
- Excelテンプレートは日本語版のみ

## 第10章 参考SQLイメージ

### 10.1 在庫検索メインクエリ（概要）

```sql
-- 基準日在庫の算出
SELECT
    T1.取引先コード,
    T1.品目コード,
    T3.主管組織コード AS 組織コード,
    T4.取引先名 AS 組織名,
    T5.予備項目08 AS 取引先名,
    T6.品目名,
    T6.区分_木プラ鉄 AS 品目種別,
    -- 実績済在庫数 = 基準日在庫 + 未承認入庫 - 未承認出庫
    (T1.現在在庫数
     - COALESCE(SUB_ACTUAL.入出庫実績合計, 0)
     + COALESCE(SUB_UNAPPROVED.未承認入庫, 0)
     - COALESCE(SUB_UNAPPROVED.未承認出庫, 0)
    ) AS 実績済在庫数
FROM
    現在在庫 T1
    INNER JOIN 取引先基本マスタ T3 ON T1.取引先コード = T3.取引先コード
    LEFT JOIN 取引先_共通 T4 ON T3.主管組織コード = T4.取引先コード
    LEFT JOIN 取引先会計 T5 ON T1.取引先コード = T5.取引先コード
    INNER JOIN 品目マスタ T6 ON T1.品目コード = T6.品目コード
    -- 基準日翌日～業務日付の入出庫実績（減算用）
    LEFT JOIN (
        SELECT 取引先コード, 品目コード, SUM(数量) AS 入出庫実績合計
        FROM 入出庫実績
        WHERE 在庫計上日 BETWEEN :基準日翌日 AND :業務日付
        GROUP BY 取引先コード, 品目コード
    ) SUB_ACTUAL ON T1.取引先コード = SUB_ACTUAL.取引先コード
                 AND T1.品目コード = SUB_ACTUAL.品目コード
WHERE
    -- 検索条件（AND結合）
    (:組織コード IS NULL OR T3.主管組織コード = :組織コード)
    AND (:取引先コード IS NULL OR T1.取引先コード = :取引先コード)
    AND (:取引先名 IS NULL OR T5.予備項目08 LIKE '%' || :取引先名 || '%')
    -- 品目コード条件（単一 or 複数OR）
    AND (
        CASE
            WHEN :品目コード IS NOT NULL THEN T1.品目コード = :品目コード
            ELSE (
                (:品目コード1 IS NULL OR T1.品目コード = :品目コード1)
                OR (:品目コード2 IS NULL OR T1.品目コード = :品目コード2)
                OR (:品目コード3 IS NULL OR T1.品目コード = :品目コード3)
                OR (:品目コード4 IS NULL OR T1.品目コード = :品目コード4)
            )
        END
    )
ORDER BY
    -- 優先表示組織を先頭
    CASE WHEN T3.主管組織コード IN (SELECT 優先組織コード FROM コードマスタ WHERE ...) THEN 0 ELSE 1 END,
    T3.主管組織コード,
    T1.取引先コード,
    T1.品目コード
```

> **注記**：上記は参考イメージであり、実際の物理テーブル名・カラム名は設計時に確定する。DB Fluteの外出しSQLとして実装する。
